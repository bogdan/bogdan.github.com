Intuitive approach

I don't think I just know

  • Takes almost no time
  • Has High Confidence
  • Always Feels Right

Problems of Intuitive Decision

  • Requires Experience
  • Hard to communicate
  • Needs respect or trust from the team
  • Discussion is difficult
  • Applicable by people with tamed ego

Naive approach

Comes when the criteria

isn't met

Typical Naive approach

  • Name Decisions
  • List Pros and Cons
  • Handle discussion
  • Choose something

Example

Which database is best for project A?

  • MySQL
  • PostgreSQL
  • MongoDB
  MySQL PostgreSQL MongoDB
Pros
  • Has support for Google BigQuery
  • Team has Scaling Experience
  • Amazon RDS Support
  • Faster performance
  • Concurrent schema update out of the box
  • Schema updates are not required
  • JSON structures support
Cons
  • The slowest on trivial operations
  • "EXPLAIN" statement is almost useless
  • Little advanced scaling tools
  • A process can crash randomly
  • Memory consumption is unpredictable
  • Weaker query language

Unanswered Questions

  • If MongoDB process can crash, how stable are other solutions?
  • If "EXPLAIN" statement is useless for MySQL, does it mean that it is useful anywhere else?
  • If Amazon RDS supports MySQL, does it support others?

Downsides of Pros/Cons approach

  • Mostly random statements about each solution
  • More solutions can not be properly compared
  • Hard to "load" all the statement to your head
  • Can make a decision political

What we need

  • All solutions are evaluated by the same criteria
  • Visually digestible
  • All parties are heard

Introducing Criteria

  • All pros and cons should be assigned to a criteria.
  • Criteria should be applicable to all solutions.
  • Otherwise it is a bad criteria.

Example

What is a criteria for each of the following statements:
  • "EXPLAIN" statement is almost useless
    • Query Optimization Tools
  • Amazon RDS support
    • Hosting Services
  • Process crashes randomly
    • Process Stability

MySQL vs PostgreSQL vs MongoDB

  • Process stability

  • Performance on trivial operations

When to apply?

  • More than 2 variants
  • No ability to change your mind
  • Long term consequences
  • No team consensus

How to apply?

  • Make sure all variants are in
  • Make sure criteria are applicable to all solutions
  • Colorize cells for better high level overview
  • It can be that good to show it to non-technical people
